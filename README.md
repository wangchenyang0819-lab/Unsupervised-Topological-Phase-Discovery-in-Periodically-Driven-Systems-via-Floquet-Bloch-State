# Floquet Topological Classification for Class A (2D)

This folder contains MATLAB code for unsupervised clustering of Floquet topological phases in symmetry class A (2D). The method constructs a kernel from the flattened Floquet operator in momentum‑time space, then applies diffusion maps and hierarchical clustering to identify topologically distinct phases.

## Files

| File | Description |
|------|-------------|
| `TwoAdiffusion.m` | Main clustering script. Builds the kernel similarity matrix between different parameter samples, runs diffusion map, performs PCA, and saves the clustering result (`A1.mat`). |
| `blochA.m` | Generates Bloch vector data for a single parameter configuration (either trivial or non‑trivial). Saves the linking‑point coordinates as `trivallink.mat` or `nontrivallink.mat`. |
| `twoDAinvariants.m` | Computes the theoretical Floquet topological invariants (winding numbers for the 0‑gap and π‑gap) for class A. Used to validate the clustering results. |
| `fig2.m` | (Not provided here) Plotting script that reads `A1.mat` and the `*link.mat` files to produce Fig. 2 in the paper. |

## Data files

- `A1.mat` – Generated by `TwoAdiffusion.m`. Contains the diffusion map eigenvalues `e` and the PCA scores `score`.
- `trivallink.mat` / `nontrivallink.mat` – Generated by `blochA.m`. Contain the (kx, ky, t) coordinates of red/blue linking points used for visualizing the Bloch vector orientation.
- `topological_invariants.mat` (optional) – Generated by `twoDAinvariants.m`. Contains the computed winding numbers for each parameter sample.

## Requirements

- MATLAB (R2020b or later recommended)
- Parallel Computing Toolbox (used in `blochA.m` – `parfor` loop)

## Usage

### 1. Generate the clustering result

Run `TwoAdiffusion.m` to:
- Define a set of 30 samples with varying hopping amplitude `J` (from 0 to 3π/T).
- For each sample, construct the Floquet effective Hamiltonian, compute the flattened Floquet operator, and evaluate the kernel matrix.
- Perform diffusion map and PCA.
- Save `A1.mat` containing eigenvalues `e` and PCA scores `score`.

```matlab
>> TwoAdiffusion
2. Compute theoretical topological invariants
Run twoDAinvariants.m to calculate the winding numbers for the 0‑gap and π‑gap for each parameter sample. The script uses the piecewise‑constant driving protocol defined in the paper and evaluates the 3D winding number via a discrete version of the Chern‑Simons form. The results can be used to validate the clusters obtained from TwoAdiffusion.m.

matlab
>> twoDAinvariants
The script produces a figure showing the invariants as a function of the parameter, and optionally saves the winding numbers to a .mat file.

3. Generate Bloch vector data for a specific phase
Edit blochA.m to select a sample index f:

f = 4 → trivial phase

f = 27 → non‑trivial phase

Then run the script to compute the time‑evolved Floquet‑Bloch states and extract linking points in (kx, ky, t) space. Results are saved as trivallink.mat or nontrivallink.mat.

matlab
>> blochA
4. Reproduce the paper figures
Run fig2.m (not included in this README) to load the saved .mat files and produce the phase diagram, eigenvalue plot, and Bloch vector visualizations as shown in the manuscript.

Key parameters
Parameter	Default	Description
m	30	Number of samples (different J values)
N	40	Grid points per momentum direction (kx, ky)
Nt	40	Grid points in time
omg	7	Number of quasi‑energy modes (controls Floquet matrix size)
epsilon	0.1	Kernel width parameter
l	10	(Unused in current version)
Output
A1.mat:

e: eigenvalues of the diffusion map (used to determine number of clusters via spectral gap)

score: first three principal components for visualisation

*link.mat:

redPoints, bluePoints: coordinates where the Bloch vector points to specific directions, indicating topological linking.

topological_invariants (if saved):

2×n_points matrix where the first row gives the 0‑gap winding number and the second row gives the π‑gap winding number for each parameter sample.

Notes
The code uses a truncated harmonic expansion up to the 5th order (H5, Hfu5). This is sufficient for the driving protocol studied in the paper.

The kernel constant epsilon should be chosen small enough to resolve topological differences but not so small that it separates topologically identical samples. A value of 0.1 works for the 2D class A system.

For other symmetry classes (AIII, D), please refer to the corresponding subfolders.
