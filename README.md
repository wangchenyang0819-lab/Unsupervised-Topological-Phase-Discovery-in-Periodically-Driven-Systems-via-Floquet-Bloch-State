# Unsupervised-Topological-Phase-Discovery-in-Periodically-Driven-Systems-via-Floquet-Bloch-State
Code for the paper Unsupervised Topological Phase Discovery in Periodically Driven Systems via Floquet-Bloch State
Overview
This repository contains MATLAB codes for calculating and visualizing topological phases, Bloch vectors, and performing cluster analysis in various systems. The project includes three main sets of programs that generate different types of topological data and visualizations.
Project Structure
Set 1: A-Type System Analysis
A1.mat​ - Data file containing eigenvalues and PCA scores (generated by running the clustering programs)
blochA.m​ - Computes Bloch vectors for the A-type system
fig2.m​ - Generates Figure 2 from the paper/results
nontrivallink.mat​ - Data file for non-trivial link configurations
trivallink.mat​ - Data file for trivial link configurations
TwoADiffusion.m​ - Runs the clustering/diffusion algorithm for A-type systems
twoDAinvariants.m​ - Computes topological invariants for 2D A-type systems
Set 2: AIII-Type System Analysis
Alll.mat​ - Data file for AIII system (contains eigenvalues, scores, topological phase data)
Alltopushu.m​ - Computes topological invariants and generates phase diagrams for AIII systems
blochAlll.m​ - Computes Bloch vectors for AIII-type systems
fig1.m​ - Generates Figure 1 from the paper/results
kernelAIII11.mat​ - Kernel data file 1 for AIII system (generated by Bloch vector calculations)
kernelAIII31.mat​ - Kernel data file 2 for AIII system (generated by Bloch vector calculations)
Set 3: D-Type System Analysis
bloch-d.m​ - Computes Bloch vectors for D-type systems
D.mat​ - Data file for D-type system (contains eigenvalues, scores, topological phase data)
diffusion-d.m​ - Runs the clustering/diffusion algorithm for D-type systems
fig3.m​ - Generates Figure 3 from the paper/results
kernelD0fu1.mat​ - Kernel data file 1 for D-type system
kernelD151.mat​ - Kernel data file 2 for D-type system
Data Generation Pipeline
Required MATLAB Data Files
Three types of .matdata files are required for running the programs:
Kernel files​ (kernel*.mat):
Contain: xx, yy, zz(Bloch vector components)
Generated by: blochA.m, blochAlll.m, bloch-d.m
Clustering result files​ (A1.mat, Alll.mat, D.mat):
Contain: e(eigenvalues), score(PCA scores), tpp(topological phase data), B(parameter coordinates), f(cluster labels)
Generated by: TwoADiffusion.m,AIIItopushu.m, diffusion-d.m
Link data files​ (*link.mat):
Contain: redPoints, bluePoints, N(grid size)
Generated by: blochA.m 

Generate Kernel Data
   └── Run Bloch vector programs:
       • blochA.m      → trivallink.mat, nontrivallink.mat
       • blochAlll.m   → kernelAIII11.mat, kernelAIII31.mat
       • bloch-d.m     → kernelD0fu1.mat, kernelD151.mat

2. Run Clustering Analysis
   └── Execute diffusion programs:
       • TwoADiffusion.m  → A1.mat 
       • Alltopushu.m     → Alll.mat
       • diffusion-d.m    → D.mat

3. Generate Figures
   └── Run visualization programs:
       • fig1.m  → Figure 1
       • fig2.m  → Figure 2
       • fig3.m  → Figure 3

Program Functions
Bloch Vector Calculation Programs
blochA.m, blochAlll.m, bloch-d.m:
Compute time-dependent Bloch vectors for different symmetry classes
Generate kernel data files containing xx, yy, zz components
Clustering/Diffusion Programs
TwoADiffusion.m, Alltopushu.m, diffusion-d.m:
Perform unsupervised clustering on parameter spaces
Calculate similarity matrices between different parameter points
Generate eigenvalues and PCA scores for dimensionality reduction
Produce topological phase diagrams
Topological Invariant Calculation
twoDAinvariants.m, Alltopushu.m:
Compute topological invariants (Chern numbers, winding numbers)
Generate topological phase diagrams
Classify different topological phases
Visualization Programs
fig1.m, fig2.m, fig3.m:
Generate publication-quality figures
Visualize Bloch vectors, phase diagrams, and clustering results
Combine multiple subplots into composite figures
Usage Instructions
Prerequisites
MATLAB R2018b or later
Statistics and Machine Learning Toolbox
Parallel Computing Toolbox (recommended for faster computation)
Step-by-Step Execution
First, generate kernel data:
% For A-type systems
blochA;

% For AIII-type systems  
blochAlll;

% For D-type systems
bloch-d;
Then, run clustering analysis:
% For A-type systems
TwoADiffusion;

% For AIII-type systems
Alltopushu;

% For D-type systems
diffusion-d;
inally, generate figures:
% Generate all figures
fig1;
fig2; 
fig3;
Expected Outputs
Kernel files: Contain 3D vector field data for Bloch vectors
Clustering files: Contain eigenvalues, PCA scores, and phase classifications
Figures: Publication-ready visualizations of results
File Dependencies
Data Files Generation:
  Kernel Files (.mat) ← bloch*.m programs
  Clustering Files (.mat) ← diffusion*/Alltopushu.m programs
  Link Files (.mat) ← Intermediate results from topological calculations

Figure Generation:
  fig*.m ← Requires corresponding .mat data files
Computational Notes
The Bloch vector calculations use Floquet theory with multiple harmonics
Clustering algorithms use similarity matrices based on wavefunction overlaps
Topological invariant calculations use both Chern numbers and winding numbers
Parallel processing is implemented for parameter space exploration
